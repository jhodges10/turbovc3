# Compatibility, limits, and fixture provenance

This document describes the `0.1.x` compatibility contract. Release artifacts include a generated JSON report with
the exact packed file list, public API-report hashes, peer/runtime requirements, distribution tag, and tarball hash.

## Browser and runtime support

- CI decodes in the current Playwright Chromium, Firefox, and WebKit engines on Node.js 22 runners.
- Chromium, Firefox, and WebKit exercise synchronous native decode, ordinary packet workers, cross-origin-isolated
  shared-row workers, TypeScript fallback, and Canvas2D output. WebGPU remains capability-detected and is not yet a
  cross-browser release guarantee.
- Node.js 22+ is supported through `@jhodges10/turbovc3/node`. Deno 2.8.3 and Bun 1.3.8 receive direct-ESM decode,
  injected-worker, fallback, and teardown smoke coverage.
- Shared memory requires COOP/COEP isolation. Other worker, WASM, TypeScript, Canvas2D, and audio paths do not.

## Mediabunny

`mediabunny` is a `^1.50.8` peer dependency and an exact `1.50.8` development dependency. CI runs the compatibility
preflight and a real DNx decode against both 1.50.8 and the newest published release. DNx recognition currently uses
a guarded extension shim; registration fails before changing global state if the required extension APIs disappear.

## Input safety limits

MXF opens accept lower caller-provided limits. Defaults cap a metadata value at 64 MiB, metadata sets at 100,000,
KLV packets and output packets at 2,000,000 each, tracks at 1,024, resynchronization work at 16 MiB, dimensions at
16,384×16,384, and a frame at 268,435,456 pixels. The native Zig ABI separately validates its published packet,
row, frame, macroblock-width, and row-count capacities before any memory access.

Treat media as untrusted input. CI includes deterministic malformed DNx/MXF corpora, mutation smoke tests, C
ASan/UBSan, Zig Debug safety checks, and typed worker-error propagation, but this experimental package has not had an
independent security audit. See `SECURITY.md` for private reporting.

## Fixture provenance

`tests/fixtures/manifest.json` records every required artifact's generation script, format/profile metadata, and
SHA-256 checksum. Redistributable required media and compressed pixel oracles are generated by FFmpeg 8.x through
`scripts/generate-ci-fixtures.sh`. CI verifies every manifest hash before comparing TypeScript, Emscripten, Zig, and
FFmpeg output. The external FFmpeg FATE 12-bit DNxHR sample is intentionally opt-in and is not redistributed here.

Coverage claims distinguish implemented parsing from required oracle evidence. Genuine adaptive MBAFF, alpha,
low-latency alpha, CID 1256, 12-bit 4:4:4, non-48-kHz MXF PCM, AES3/big-endian audio, and files from additional NLE
vendors remain explicitly unverified until suitable redistributable fixtures are available.
